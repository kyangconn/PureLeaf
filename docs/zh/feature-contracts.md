# 功能契约

本文给人和 AI 看，用来判断一个功能应该改哪里、不能怎么改。

## 新建空白项目

输入：

- 父目录
- 项目名称

行为：

- 创建项目目录。
- 缺少 `main.tex` 时写入默认模板。
- 注册项目。
- 更新最近项目。
- 打开 Editor。

不要：

- 从 `MainWindow` 直接写多种存储。
- 同时写 QSettings 和 SQLite 作为两份真相。
- 静默覆盖已有非空目录。

## 打开本地文件夹

输入：

- 本地目录路径

行为：

- 校验目录存在。
- 若未登记，则注册为项目或最近项目。
- 更新最近项目。
- 打开 Editor。

不要：

- 因 Git 检测失败阻止打开。
- 扫描/索引大目录时阻塞 UI 太久。
- 复制用户文件到 app 数据目录。

## 最近项目

行为：

- 按最近打开时间排序。
- 显示项目名、路径、字符数、Git 状态。
- 移除最近项目只影响列表，不删除磁盘内容。

不要：

- 把“最近项目移除”做成项目删除。
- 在 UI 组件里维护另一套持久化列表。

## Git 状态

行为：

- 只读检测。
- 支持 Clean、Modified、Conflict、NoRepository、Unknown。
- 检测失败返回 Unknown。

不要：

- 在 Home 自动执行会修改仓库的 Git 命令。
- 因 Git 不可用阻断项目打开。

## 设置

行为：

- 保存用户偏好。
- 不保存项目业务数据。

不要：

- 用 QSettings 存项目元数据、文件索引、历史快照。
